# Relatório de Segurança — Olympus Frontend (Athenio.ai)

**Data:** 2026-04-06
**Escopo:** Todo o projeto (src/)
**Auditor:** Security Auditor Agent (SVVA Framework)

## Sumário

| Severidade | Quantidade |
|-----------|-----------|
| CRÍTICO | 1 |
| ALTO | 2 |
| MÉDIO | 3 |
| BAIXO | 1 |

## Top 3 Riscos

1. **[CRÍTICO]** Rotas admin sem verificação de role — qualquer user autenticado acessa /admin/*
2. **[ALTO]** Sem security headers (CSP, X-Frame-Options, HSTS)
3. **[ALTO]** Login leaka mensagens de erro do backend

## Findings

### 1. [CRÍTICO] Sem RBAC nas rotas admin
- **Arquivo:** `src/app/(authenticated)/layout.tsx:5-13`, `src/components/sidebar.tsx:35-38`
- **OWASP:** A01 - Broken Access Control
- **Fix:** Adicionar `role` ao AuthUser + guard no admin layout + verificar role nas server actions

### 2. [ALTO] Sem security headers
- **Arquivo:** `next.config.ts`
- **OWASP:** A05 - Security Misconfiguration
- **Fix:** Adicionar X-Frame-Options, CSP, HSTS, X-Content-Type-Options no next.config.ts

### 3. [ALTO] Login leaka erro do backend
- **Arquivo:** `src/app/login/actions.ts:16-18`
- **OWASP:** A07 - Auth Failures
- **Fix:** Retornar mensagem genérica "Email ou senha inválidos"

### 4. [MÉDIO] deleteChatSession e updateCalendarConfig leakam erros
- **Arquivo:** `src/app/(authenticated)/conversas/actions.ts:14-16`, `src/app/(authenticated)/configuracoes/actions.ts:17`
- **Fix:** Usar padrão SAFE_ERRORS em todas as actions

### 5. [MÉDIO] tryRefresh não persiste novo refresh token
- **Arquivo:** `src/lib/services/auth-service.ts:80-98`
- **Fix:** Persistir ambos tokens (ou aceitar que middleware é o caminho principal)

### 6. [MÉDIO] Upload de PDF valida apenas File.type (spoofável)
- **Arquivo:** `src/app/(authenticated)/admin/usuarios/actions.ts:30-31`
- **Fix:** Adicionar validação de magic bytes (%PDF-)

### 7. [BAIXO] updateCalendarConfig sem validação de input
- **Arquivo:** `src/app/(authenticated)/configuracoes/actions.ts:7-20`
- **Fix:** Adicionar Zod schema validation

### 8. [INFO] .env existe mas sem dados sensíveis
- Apenas NEXT_PUBLIC_API_URL=localhost. Gitignored.

## Positivos Confirmados

- Zero XSS (sem dangerouslySetInnerHTML, sem eval)
- Zero hardcoded secrets
- CSRF protegido (server actions nativos do Next.js)
- Cookies httpOnly + sameSite + secure em produção
- Sem SQL injection (frontend não acessa banco)
